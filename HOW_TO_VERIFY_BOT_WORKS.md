# Как проверить, что бот работает правильно 🎵

## 📊 Команды для проверки работы бота

### 1. `/status` - Проверка статуса мониторинга
Показывает:
- Какие страницы отслеживаются
- Когда была последняя проверка
- Интервал проверки
- Статус работы бота

**Пример использования:**
```
/status
```

**Что вы увидите:**
```
📊 Статус мониторинга:

🔗 memobar.ru
   URL: https://memobar.ru/
   📅 Последняя проверка: 2024-10-04 11:15:30
   ✅ Статус: Активен

⏰ Интервал проверки: 5 минут
🤖 Бот работает и мониторит ваши страницы!

💡 Когда на сайтах появятся новые события, вы получите уведомления.
```

### 2. `/simulate` - Симуляция нового события
Отправляет тестовое уведомление, чтобы показать, как будут выглядеть реальные уведомления.

**Пример использования:**
```
/simulate
```

**Что произойдет:**
- Бот отправит тестовое изображение с событием
- Покажет формат будущих уведомлений
- Подтвердит, что система уведомлений работает

### 3. `/test <URL>` - Тестирование парсинга
Проверяет, как бот парсит конкретную страницу.

**Пример использования:**
```
/test https://memobar.ru/
```

**Что вы увидите:**
- Количество найденных событий
- Примеры событий с изображениями
- Статистику по изображениям
- Тестовое изображение

### 4. `/list` - Список отслеживаемых страниц
Показывает все страницы, которые бот мониторит.

**Пример использования:**
```
/list
```

## 🔍 Как понять, что бот работает

### ✅ Признаки правильной работы:

1. **Команда `/status` показывает активные страницы**
2. **Время последней проверки обновляется каждые 5 минут**
3. **Команда `/simulate` отправляет тестовое уведомление**
4. **Команда `/test` находит события на странице**

### 📱 Что происходит в фоне:

1. **Каждые 5 минут** бот проверяет все отслеживаемые страницы
2. **Сравнивает содержимое** с предыдущей версией
3. **При обнаружении изменений** отправляет уведомления
4. **Логирует все действия** для отладки

## 🚨 Что делать, если что-то не работает

### Проблема: Команда `/status` показывает "Никогда" в последней проверке
**Решение:** Проверьте логи бота на наличие ошибок

### Проблема: Команда `/simulate` не работает
**Решение:** Убедитесь, что у вас есть отслеживаемые страницы

### Проблема: Команда `/test` не находит события
**Решение:** Возможно, нужно настроить селекторы для сайта

## 💡 Советы по использованию

1. **Регулярно проверяйте `/status`** - это покажет, что бот активен
2. **Используйте `/simulate`** для проверки формата уведомлений
3. **Тестируйте новые сайты** командой `/test` перед добавлением
4. **Следите за логами** - они показывают активность бота

## 🎯 Ожидание реальных событий

Когда на сайтах появятся новые события:
- Бот автоматически их обнаружит
- Отправит уведомления в том же формате, что и `/simulate`
- Покажет изображения афиш (если есть)
- Включит ссылки на события

**Помните:** Бот работает 24/7 и проверяет сайты каждые 5 минут!
