# Руководство по работе с графическими афишами 🖼️

## Особенности страниц с графическими афишами

Некоторые сайты концертных площадок размещают только графические афиши без текстовых описаний событий. Для таких страниц бот использует специальную логику извлечения изображений.

## Как работает извлечение изображений

### 1. Автоматическое определение
Бот автоматически определяет сайты с графическими афишами по ключевым словам в URL:
- `poster`, `afisha`, `афиша`, `posters`, `events`, `concerts`

### 2. Фильтрация изображений
Бот исключает служебные изображения:
- Логотипы, иконки, аватары
- Баннеры, заголовки, подвалы
- Социальные сети (Facebook, Twitter, Instagram, VK)
- Загрузочные индикаторы, плейсхолдеры
- Изображения меньше 100x100 пикселей

### 3. Извлечение данных
Для каждого найденного изображения бот извлекает:
- **URL изображения** - полная ссылка на афишу
- **Заголовок** - из атрибутов `alt` или `title`, или из родительского элемента
- **Ссылку на событие** - если изображение находится внутри ссылки
- **Уникальный хеш** - для отслеживания изменений

## Команды для работы с графическими афишами

### `/images <URL>` - Тестирование извлечения изображений
```
/images https://example.com/posters
```

Эта команда:
- Показывает количество найденных изображений-афиш
- Выводит информацию о первых 5 изображениях
- Отправляет пример афиши как изображение
- Помогает понять, правильно ли бот распознает афиши

### `/test <URL>` - Общий тест парсинга
```
/test https://example.com/posters
```

Эта команда показывает общие результаты парсинга, включая изображения.

## Примеры использования

### Тестирование страницы с афишами
1. Отправьте: `/images https://example.com/concerts`
2. Бот покажет найденные изображения
3. Если изображения найдены, добавьте страницу: `/add https://example.com/concerts`

### Добавление страницы для мониторинга
1. Используйте команду `/add` или просто отправьте ссылку
2. Бот автоматически определит тип страницы
3. Начнет мониторинг изображений

## Настройка для конкретных сайтов

Если бот не правильно распознает афиши на вашем сайте, можно настроить специальные селекторы в файле `site_configs.py`:

```python
'your-site.com': {
    'name': 'Ваш сайт',
    'event_selector': '.poster-item, .afisha-card',
    'title_selector': '.poster-title, img[alt]',
    'date_selector': '',  # Дата в изображении
    'link_selector': 'a',
    'image_selector': 'img.poster-image',
    'description': 'Сайт с графическими афишами'
}
```

## Особенности уведомлений

Для страниц с графическими афишами уведомления содержат:
- 🎵 Название события (из alt/title или общее)
- 🖼️ Само изображение-афишу
- 🔗 Ссылку на событие (если есть)

## Устранение проблем

### Изображения не найдены
- Проверьте размеры изображений (должны быть больше 100x100)
- Убедитесь, что изображения не помечены как служебные
- Используйте команду `/images` для диагностики

### Слишком много ложных срабатываний
- Добавьте исключения в `_is_event_image()` метод
- Настройте специальные селекторы для сайта

### Изображения не отправляются
- Проверьте доступность URL изображений
- Убедитесь, что изображения не требуют авторизации
- Проверьте размер изображений (Telegram имеет ограничения)
