# 🔍 Руководство по сканированию

## 🎯 Зачем нужны команды сканирования?

Иногда нужно убедиться, что бот действительно работает и сканирует ваши сайты. Новые команды позволяют:

- ✅ **Принудительно просканировать** все ссылки прямо сейчас
- ✅ **Протестировать парсинг** новой ссылки перед добавлением
- ✅ **Проверить статус** последних проверок
- ✅ **Убедиться** что мониторинг активен

## 📱 Новые команды

### 🔍 `/scan` - Принудительное сканирование

**Что делает:** Сканирует все ваши ссылки прямо сейчас

**Когда использовать:**
- Хотите проверить ссылки немедленно
- Подозреваете что бот не работает
- Нужно убедиться что мониторинг активен

**Пример:**
```
/scan
```

**Результат:**
```

📡 Сканирование 1/2: https://example.com/events...

📅 Найдено событий: 5
🆕 Новых событий: 1
🔄 Последняя проверка: 04.10.2025 15:30:45

📡 Сканирование 2/2: https://music-venue.ru/concerts...
📊 Результат сканирования:
🔗 https://music-venue.ru/concerts
📅 Найдено событий: 3
🆕 Новых событий: 0
🔄 Последняя проверка: 04.10.2025 15:30:50

✅ Сканирование всех ссылок завершено!
```

### 🔍 `/scan <номер>` - Сканирование конкретной ссылки

**Что делает:** Сканирует только одну ссылку по номеру

**Когда использовать:**
- Хотите проверить только одну ссылку
- Нужно быстро протестировать конкретный сайт
- Отладка проблем с парсингом

**Пример:**

/scan 1
```

**Результат:**

### 🧪 `/test <ссылка>` - Тестирование парсинга

**Что делает:** Тестирует парсинг ссылки без добавления в мониторинг

**Когда использовать:**
- Хотите проверить парсинг перед добавлением
- Отладка проблем с конкретным сайтом
- Проверка работы селекторов

**Пример:**

/test https://new-site.com/events
```

## 🎯 Сценарии использования

### 🔍 Проверка работы бота

1. **Добавьте ссылку:**
   ```
   /add https://example.com/events
   ```

2. **Проверьте статус:**
   ```
   /status
   ```

3. **Принудительно просканируйте:**
   ```
   /scan
   ```

4. **Посмотрите логи:**
   ```
   /logs
   ```

### 🧪 Тестирование новой ссылки

1. **Протестируйте парсинг:**
   ```
   /test https://new-site.com/events
   ```

2. **Если парсинг работает, добавьте ссылку:**
   ```
   /add https://new-site.com/events
   ```

3. **Проверьте что ссылка добавлена:**
   ```
   /list
   ```

### 🐛 Отладка проблем

1. **Проверьте статус всех ссылок:**
   ```
   /status
   ```

2. **Просканируйте проблемную ссылку:**
   ```
   /scan 1
   ```

3. **Посмотрите логи:**
   ```
   /logs
   ```

## 💡 Советы

- **Используйте `/test`** перед добавлением новых ссылок
- **Регулярно проверяйте `/status`** чтобы убедиться что мониторинг активен
- **Используйте `/scan`** если подозреваете что бот не работает
- **Смотрите `/logs`** чтобы понять когда последний раз проверялись ссылки

## ⚠️ Важные моменты

- **Принудительное сканирование** может занять время
- **Тестирование** не добавляет ссылку в мониторинг
- **Логи** показывают только ваши ссылки
- **Статус** обновляется в реальном времени
