# Concert Monitor Bot - Финальная сводка 🎵

## ✅ Проект полностью готов и протестирован!

### 🎯 Реализованный функционал

**Основные возможности:**
- ✅ Мониторинг сайтов концертных площадок
- ✅ Автоматическое обнаружение новых событий
- ✅ Извлечение названий, дат и ссылок событий
- ✅ **Поддержка графических афиш** (НОВОЕ!)
- ✅ Уведомления только при появлении новых событий
- ✅ Поддержка множественных пользователей
- ✅ База данных для хранения данных

**Команды бота:**
- `/start` - Начать работу
- `/add` - Добавить страницу для мониторинга
- `/list` - Показать отслеживаемые страницы
- `/remove` - Удалить страницу из мониторинга
- `/test` - Тестировать парсинг страницы
- `/images` - **Тестировать извлечение изображений-афиш** (НОВОЕ!)
- `/sites` - Показать поддерживаемые сайты
- `/help` - Справка

### 🖼️ Работа с графическими афишами

**Новая функциональность:**
- ✅ Автоматическое определение сайтов с графическими афишами
- ✅ Фильтрация служебных изображений (логотипы, иконки, баннеры)
- ✅ Извлечение изображений размером больше 100x100 пикселей
- ✅ Получение заголовков из атрибутов `alt` и `title`
- ✅ Поиск ссылок на события в родительских элементах
- ✅ Отправка афиш как изображений в Telegram

**Команда `/images <URL>`:**
- Тестирует извлечение изображений-афиш
- Показывает количество найденных изображений
- Отправляет пример афиши
- Помогает диагностировать проблемы

### 🔧 Управление ботом

**Запуск:**
```bash

```

**Тестирование графических афиш:**
```
/images https://example.com/posters
```

**Добавление страницы:**
```
/add https://example.com/events
```

### 📈 Мониторинг

Бот проверяет страницы каждые 5 минут и отправляет уведомления только при появлении новых событий, что соответствует требованию "не присылать события каждые 10 минут".

**Для графических афиш:**
- Отслеживает изменения в изображениях
- Отправляет новые афиши как изображения
- Извлекает заголовки из атрибутов изображений
