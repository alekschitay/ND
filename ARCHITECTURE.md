# Архитектура Concert Monitor Bot

## Общая схема

```
👤 Пользователь
    ↓ (Telegram API)
📱 Telegram Bot (bot.py)
    ↓
🗄️ База данных (database.py) ← 📊 Отслеживаемые страницы
    ↓
🔍 Мониторинг (monitor.py) ← ⚙️ Конфигурации (site_configs.py)
    ↓
🌐 HTTP запросы (aiohttp) + 📄 Парсинг HTML (BeautifulSoup)
    ↓
🎵 События + 🖼️ Афиши
    ↓
📨 Уведомления → 📱 Telegram Bot → 👤 Пользователь
```

## Компоненты

### 1. bot.py - Основной модуль
- Обработка команд Telegram
- Управление пользователями
- Отправка уведомлений
- Координация работы всех модулей

### 2. database.py - База данных
- SQLite база данных
- Таблицы: users, monitored_pages, events
- Асинхронные операции с БД

### 3. monitor.py - Мониторинг
- Загрузка веб-страниц
- Парсинг HTML контента
- Извлечение событий
- Вычисление хешей для отслеживания изменений

### 4. site_configs.py - Конфигурации
- CSS селекторы для разных сайтов
- Специфичные настройки парсинга
- Поддержка популярных концертных площадок

### 5. config.py - Настройки
- Конфигурация бота
- Интервалы мониторинга
- Общие селекторы

## Поток данных

1. **Добавление страницы**: Пользователь → bot.py → database.py
2. **Мониторинг**: monitor.py → HTTP запрос → Парсинг → Сравнение хешей
3. **Обнаружение изменений**: Новые события → database.py → bot.py → Уведомления
4. **Отправка уведомлений**: bot.py → Telegram API → Пользователь

## Особенности

- **Асинхронность**: Все операции выполняются асинхронно
- **Устойчивость**: Обработка ошибок и повторные попытки
- **Масштабируемость**: Поддержка множественных пользователей и сайтов
- **Гибкость**: Легкое добавление новых сайтов через конфигурации
