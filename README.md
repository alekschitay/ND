# Concert Monitor Bot 🎵

Телеграм-бот для мониторинга сайтов концертных площадок и уведомления о новых событиях.

## Возможности

- 🎤 Отслеживание новых событий на сайтах концертных площадок
- 📅 Автоматическое извлечение дат событий
- 🔗 Получение ссылок на события
- 🖼️ Отправка графических афиш
- ⚡ Уведомления только при появлении новых событий
- 🔍 Тестирование парсинга страниц
- 📊 Поддержка множественных сайтов

## Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd concert-monitor-bot
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка переменных окружения

Скопируйте файл `.env.example` в `.env` и заполните его:

```bash
cp .env.example .env
```

Отредактируйте файл `.env`:

```env
BOT_TOKEN=your_telegram_bot_token_here
```

### 4. Получение токена бота

1. Найдите [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен в файл `.env`

## Запуск

```bash
python bot.py
```

## Использование

### Команды бота

- `/start` - Начать работу с ботом
- `/add` - Добавить страницу для мониторинга
- `/list` - Показать все отслеживаемые страницы
- `/remove` - Удалить страницу из мониторинга
- `/test` - Тестировать парсинг страницы
- `/help` - Показать справку

### Добавление страницы для мониторинга

1. Используйте команду `/add`
2. Или просто отправьте ссылку на страницу

Примеры поддерживаемых сайтов:
- https://kudago.com/moscow/concerts/
- https://afisha.ru/moscow/concert/
- https://concert.ru/events/

### Тестирование парсинга

Используйте команду `/test <URL>` для проверки, как бот парсит конкретную страницу:

```
/test https://example.com/events
```

## Конфигурация

### Настройки мониторинга

В файле `config.py` можно изменить:

```python
CHECK_INTERVAL_MINUTES = 5  # Интервал проверки в минутах
MAX_EVENTS_PER_CHECK = 10   # Максимум событий за одну проверку
```

### Селекторы для сайтов

Бот автоматически определяет селекторы для популярных сайтов. Для добавления поддержки нового сайта отредактируйте `config.py`:

```python
SITE_PATTERNS = {
    'example.com': {
        'event_selector': '.event-item',
        'title_selector': '.event-title',
        'date_selector': '.event-date',
        'link_selector': 'a',
        'image_selector': 'img'
    }
}
```

## База данных

Бот использует SQLite базу данных со следующими таблицами:

- `users` - Пользователи бота
- `monitored_pages` - Отслеживаемые страницы
- `events` - Найденные события

## Логирование

Бот ведет подробные логи работы. Уровень логирования можно настроить в `bot.py`:

```python
logging.basicConfig(level=logging.INFO)
```

## Устранение неполадок

### Бот не отвечает

1. Проверьте правильность токена в `.env`
2. Убедитесь, что бот запущен
3. Проверьте логи на наличие ошибок

### Не находит события на сайте

1. Используйте команду `/test` для проверки парсинга
2. Возможно, нужно добавить специфичные селекторы для сайта
3. Проверьте, что сайт доступен и не блокирует ботов

### Ошибки базы данных

1. Убедитесь, что у бота есть права на запись в директорию
2. Проверьте, что SQLite установлен
3. При необходимости удалите файл `concert_monitor.db` для пересоздания

## Разработка

### Добавление новых селекторов

Для добавления поддержки нового сайта:

1. Определите CSS селекторы для событий
2. Добавьте паттерны в `config.py`
3. Протестируйте с помощью команды `/test`

### Расширение функциональности

Основные компоненты для расширения:

- `monitor.py` - логика парсинга страниц
- `database.py` - работа с данными
- `bot.py` - интерфейс бота
