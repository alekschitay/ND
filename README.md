# Telegram концертный мониторинг

Этот проект реализует Telegram-бота, который отслеживает страницы с афишами концертных площадок и присылает оповещение при появлении новых событий.

## Возможности

- `/watch <url>` — добавить страницу площадки для мониторинга.
- `/unwatch <url>` — остановить слежение за страницей.
- `/list` — показать список активных подписок.
- Автоматическое определение названия, даты и ссылки на событие.
- Если у события отсутствует текстовое описание, бот отправит графическую афишу (изображение) вместо текста.
- Отправка уведомлений только при появлении новых событий.

## Запуск

1. Создайте нового бота у [BotFather](https://t.me/botfather) и получите токен.
2. Скопируйте файл `.env.example` и заполните его:

   ```bash
   cp .env.example .env
   ```

3. Установите зависимости и запустите бота:

   ```bash
   python -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   python -m bot.main
   ```

## Переменные окружения

| Имя                    | Назначение                                               | Значение по умолчанию |
|------------------------|-----------------------------------------------------------|------------------------|
| `TELEGRAM_BOT_TOKEN`   | Токен Telegram-бота (обязательный параметр).             | —                      |
| `BOT_DATABASE_PATH`    | Путь до файла SQLite для хранения подписок.              | `./bot.db`             |
| `BOT_CHECK_INTERVAL`   | Интервал проверки обновлений (в секундах).               | `300`                  |
| `BOT_REQUEST_TIMEOUT`  | Таймаут HTTP-запросов к страницам площадок (в секундах). | `20`                   |

## Как это работает

1. При добавлении новой подписки бот выполняет первичное сканирование страницы и запоминает найденные события как базовую точку отсчёта. Уведомления приходят только для последующих изменений.
2. Фоновый монитор периодически скачивает страницу, извлекает из HTML события и сравнивает их с уже отправленными. Новые события немедленно отправляются в чат.
3. Для каждого события бот пытается определить название и дату. Если текст не найден, бот ищет изображение-афишу и отправляет его.

## Ограничения

- Структура сайтов афиш различается, поэтому извлечение данных работает по эвристикам. При необходимости можно добавить дополнительные правила под конкретные площадки.
- Бот хранит состояние в локальной базе данных SQLite; убедитесь, что каталогу записи доступны права на запись.
