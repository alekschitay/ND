# Concert Monitor - Мониторинг концертных площадок

Веб-сервис для мониторинга сайтов концертных площадок и уведомления о новых событиях.

## Возможности

- 🎵 Мониторинг сайтов концертных площадок каждые 10 минут
- 🔔 Уведомления в Telegram и по email о новых событиях
- 🖼️ Поддержка изображений (афиш) событий
- 🌐 Веб-интерфейс для управления мониторингом
- 📊 Статистика и история событий
- 🐳 Docker контейнеризация

## Установка и запуск

### С Docker (рекомендуется)

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd concert-monitor
```

2. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

3. Заполните настройки в `.env`:
```env
# Telegram Bot (обязательно для уведомлений)
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
TELEGRAM_CHAT_ID=your_chat_id

# Email (опционально)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your_email@gmail.com
SMTP_PASSWORD=your_app_password
EMAIL_FROM=your_email@gmail.com

# Безопасность
SECRET_KEY=your_secret_key_here
```

4. Запустите с Docker Compose:
```bash
docker-compose up -d
```

5. Откройте веб-интерфейс: http://localhost:8000

### Локальная установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Установите PostgreSQL и создайте базу данных

3. Настройте переменные окружения (см. выше)

4. Запустите приложение:
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## Настройка Telegram бота

1. Создайте бота через [@BotFather](https://t.me/botfather)
2. Получите токен бота
3. Узнайте ваш Chat ID:
   - Напишите боту `/start`
   - Перейдите по ссылке: `https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates`
   - Найдите `chat.id` в ответе

## Использование

### Добавление сайта для мониторинга

1. Откройте веб-интерфейс
2. Нажмите "Добавить сайт"
3. Заполните форму:
   - **Название**: Название площадки
   - **URL**: Полный адрес страницы с событиями
   - **CSS селектор**: Селектор для элементов событий

### Примеры CSS селекторов

- `.event-item` - для элементов с классом event-item
- `.concert-card` - для карточек концертов
- `div.event` - для div элементов с классом event
- `li[class*="event"]` - для li элементов, содержащих "event" в классе

### API Endpoints

- `GET /` - Главная страница
- `GET /api/sites` - Список сайтов
- `POST /api/sites` - Добавить сайт
- `PUT /api/sites/{id}` - Обновить сайт
- `DELETE /api/sites/{id}` - Удалить сайт
- `GET /api/events` - Список событий
- `POST /api/check/{site_id}` - Проверить сайт вручную
- `POST /api/check-all` - Проверить все сайты
- `GET /health` - Проверка состояния

## Мониторинг

Сервис автоматически проверяет все активные сайты каждые 10 минут. При обнаружении новых событий отправляются уведомления.

### Формат уведомлений

**Telegram:**
```
🎵 Новое событие!

Название: Концерт группы "Название"
Дата: 15.12.2024 20:00
Ссылка: https://example.com/event/123
Описание: Описание события...

Источник: Название площадки
```

**Email:**
HTML-письмо с изображением афиши (если доступно) и полной информацией о событии.

## Логирование

Логи выводятся в консоль и включают:
- Информацию о проверках сайтов
- Ошибки скрапинга
- Статус отправки уведомлений
- Общую статистику работы

## Безопасность

- Используйте сильный SECRET_KEY в продакшене
- Настройте HTTPS для веб-интерфейса
- Ограничьте доступ к API endpoints
- Регулярно обновляйте зависимости

## Поддержка

При возникновении проблем:
1. Проверьте логи приложения
2. Убедитесь в правильности CSS селекторов
3. Проверьте настройки уведомлений
4. Убедитесь, что сайты доступны
