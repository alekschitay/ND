# Concert Monitor - Резюме проекта

## 🎯 Цель проекта

Создать веб-сервис для автоматического мониторинга сайтов концертных площадок с уведомлениями о новых событиях.

## ✅ Выполненные требования

### Основные функции:
- ✅ **Мониторинг каждые 10 минут** - автоматическая проверка сайтов
- ✅ **Уведомления только при изменениях** - не спамит, только новые события
- ✅ **Поддержка афиш** - извлечение и отправка графических изображений
- ✅ **Гибкие селекторы** - CSS селекторы для любых сайтов
- ✅ **Формат событий** - ссылка, дата, название события

### Технические возможности:
- ✅ **Веб-интерфейс** - управление через браузер
- ✅ **API** - REST API для интеграции
- ✅ **База данных** - хранение событий и истории
- ✅ **Уведомления** - Telegram и Email
- ✅ **Docker** - контейнеризация для развертывания

## 🚀 Способы запуска

### 1. Docker (рекомендуется)
```bash
docker-compose up -d
```

### 2. Локальная установка
```bash
pip install -r requirements.txt
python init_db.py
python run.py
```

### 3. Демо версия
Откройте `demo.html` в браузере

## 🔧 Настройка

### Обязательные настройки:
- `TELEGRAM_BOT_TOKEN` - токен Telegram бота
- `TELEGRAM_CHAT_ID` - ID чата для уведомлений

### Опциональные настройки:
- `SMTP_*` - настройки email
- `DATABASE_URL` - подключение к БД
- `SECRET_KEY` - ключ безопасности

## 📊 Мониторинг сайтов

### Примеры конфигурации:
- **Крокус Сити Холл**: `.event-card`
- **Олимпийский**: `.concert-item`
- **СК Олимпийский**: `.event-block`

### Процесс мониторинга:
1. Каждые 10 минут проверка всех активных сайтов
2. Извлечение событий по CSS селектору
3. Сравнение с уже известными событиями
4. Отправка уведомлений только для новых событий
